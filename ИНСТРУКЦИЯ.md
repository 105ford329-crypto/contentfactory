# Инструкция по настройке Content Factory для n8n

## Общие требования

### Необходимые аккаунты и API ключи

| Сервис | Для чего | Как получить |
|--------|----------|--------------|
| **Apify** | Парсинг TikTok/Instagram | apify.com → Settings → API tokens |
| **OpenRouter** | LLM (Gemini + Claude) | openrouter.ai → Keys |
| **HeyGen** | AI-аватар видео | heygen.com → API Settings |
| **Creatomate** | Монтаж видео | creatomate.com → API Keys |
| **Google Cloud** | Sheets + Drive | console.cloud.google.com → OAuth 2.0 |
| **Telegram** | Уведомления | @BotFather → /newbot |
| **Threads API** | Публикация постов | Meta Developer Portal |
| **Anthropic** | Claude напрямую | anthropic.com → API Keys |

### Настройка Credentials в n8n

1. **Apify API**
   - Тип: Header Auth
   - Name: `Authorization`
   - Value: `Bearer YOUR_APIFY_TOKEN`

2. **OpenRouter API**
   - Тип: Header Auth
   - Name: `Authorization`
   - Value: `Bearer YOUR_OPENROUTER_KEY`

3. **HeyGen API**
   - Тип: Header Auth
   - Name: `X-Api-Key`
   - Value: `YOUR_HEYGEN_KEY`

4. **Creatomate API**
   - Тип: Header Auth
   - Name: `Authorization`
   - Value: `Bearer YOUR_CREATOMATE_KEY`

5. **Google Sheets / Drive**
   - Тип: OAuth2
   - Настроить через Google Cloud Console

6. **Telegram Bot**
   - Тип: Telegram API
   - Token от @BotFather

7. **Threads API**
   - Тип: Header Auth
   - Name: `Authorization`
   - Value: `Bearer YOUR_THREADS_ACCESS_TOKEN`

8. **Anthropic API** (для прямого использования Claude)
   - Тип: Anthropic API
   - API Key от anthropic.com

## Workflow 1: TikTok Life Topics

**Файл:** `workflow-1-tiktok-life-topics.json`

**Назначение:** Парсинг конкурентов, анализ трендов, генерация тем для контента

### Что делает

1. Каждые 24 часа (в 9:00) запускает парсинг
2. Параллельно собирает данные с 3 источников:
   - TikTok аккаунты (группа 1)
   - TikTok аккаунты (группа 2)
   - Instagram аккаунты
3. Анализирует тренды через Gemini Flash
4. Генерирует 5 тем через Claude Sonnet
5. Сохраняет в Google Sheets

### Настройка

**Шаг 1: Замените username конкурентов**

В нодах `Headline patterns1`, `Headline patterns2`, `Description patterns1`:

```json
"profiles": [
  "COMPETITOR_USERNAME_1",  // ← ваши конкуренты TikTok
  "COMPETITOR_USERNAME_2",
  "COMPETITOR_USERNAME_3"
]
```

```json
"usernames": [
  "INSTAGRAM_COMPETITOR_1",  // ← ваши конкуренты Instagram
  "INSTAGRAM_COMPETITOR_2"
]
```

**Шаг 2: Создайте Google таблицу**

Лист `Topics` с колонками:
| id | created_at | status | hook | body | cta | description | hashtags | full_script |

**Шаг 3: Замените ID таблицы**

В ноде `Google Sheets`:
```
YOUR_GOOGLE_SHEET_ID → ID вашей таблицы
```

**Шаг 4: Подключите credentials**

- APIFY_CREDENTIALS_ID
- OPENROUTER_CREDENTIALS_ID
- GOOGLE_SHEETS_CREDENTIALS_ID

### Результат

После запуска в таблице появятся 5 новых тем со статусом `pending`, готовых для следующего workflow.

## Workflow 2: TikTok AI Lifehacks

**Файл:** `workflow-2-tiktok-ai-lifehacks.json`

**Назначение:** Обогащение тем AI-инструментами, генерация скриптов, отправка в HeyGen

### Что делает

1. Каждые 24 часа (в 10:00) читает pending темы из Sheets
2. **Верхняя ветка:**
   - Берёт 5 тем
   - Для каждой генерирует AI-лайфхаки (Gemini)
   - Создаёт финальный скрипт (Claude)
3. **Нижняя ветка:**
   - Парсит AI-конкурентов
   - Анализирует их контент
4. Объединяет данные
5. Отправляет скрипт в HeyGen для генерации видео
6. Обновляет статус в Sheets

### Настройка

**Шаг 1: Настройте HeyGen**

1. Зайдите в HeyGen → Avatars → выберите аватар
2. Скопируйте `avatar_id`
3. Зайдите в Voices → выберите голос
4. Скопируйте `voice_id`

В ноде `HTTP Request`:
```json
"avatar_id": "YOUR_AVATAR_ID",  // ← ваш ID
"voice_id": "YOUR_VOICE_ID"     // ← ваш ID
```

**Шаг 2: Добавьте AI-конкурентов**

В нодах `Headline patterns`, `Description patterns`:
```json
"profiles": ["AI_COMPETITOR_1", "AI_COMPETITOR_2"]
"usernames": ["AI_INSTAGRAM_1"]
```

**Шаг 3: Подключите credentials**

- GOOGLE_SHEETS_CREDENTIALS_ID (та же таблица)
- OPENROUTER_CREDENTIALS_ID
- APIFY_CREDENTIALS_ID
- HEYGEN_CREDENTIALS_ID

### Результат

- Темы получают статус `video_generating`
- В колонке `video_id` появляется ID видео HeyGen
- Видео генерируется ~5-10 минут

## Workflow 3: Монтаж видео для Instagram

**Файл:** `workflow-3-video-montage-instagram.json`

**Назначение:** Загрузка видео/аудио из Drive, монтаж через Creatomate, отправка в Telegram

### Что делает

1. Каждые 24 часа (в 11:00) читает очередь `Production_Queue`
2. Фильтрует записи со статусом `ready_for_production` и форматом `voiceover_broll`
3. Параллельно загружает:
   - 4 видео из разных папок Drive
   - 1 аудио
   - 1 шрифт
4. Склеивает все URL
5. Отправляет в Creatomate для рендеринга
6. Ждёт готовности (цикл проверки)
7. Отправляет превью в Telegram
8. Обновляет статус в Sheets

### Настройка

**Шаг 1: Создайте структуру папок в Google Drive**

```
Content Factory/
├── Videos/
│   ├── Folder1/  → YOUR_VIDEO_FOLDER_ID_1
│   ├── Folder2/  → YOUR_VIDEO_FOLDER_ID_2
│   ├── Folder3/  → YOUR_VIDEO_FOLDER_ID_3
│   └── Folder4/  → YOUR_VIDEO_FOLDER_ID_4
├── Audio/        → YOUR_AUDIO_FOLDER_ID
└── Fonts/
    └── font.ttf  → YOUR_FONT_FILE_ID
```

**Шаг 2: Создайте шаблон в Creatomate**

1. Зайдите в Creatomate → Templates → Create
2. Создайте шаблон с элементами:
   - `Video-1`, `Video-2`, `Video-3`, `Video-4`
   - `Hook-Text`, `Body-Text`, `CTA-Text`
   - `Audio`
3. Скопируйте Template ID

**Шаг 3: Замените ID в workflow**

```
YOUR_VIDEO_FOLDER_ID_1..4 → ID папок с видео
YOUR_AUDIO_FOLDER_ID → ID папки с аудио
YOUR_FONT_FILE_ID → ID файла шрифта
YOUR_CREATOMATE_TEMPLATE_ID → ID шаблона
YOUR_TELEGRAM_CHAT_ID → ID чата (получить: @userinfobot)
YOUR_GOOGLE_SHEET_ID → ID таблицы
```

**Шаг 4: Создайте лист Production_Queue**

| id | status | format | hook | body | cta | description | hashtags | full_script | final_video_url | rendered_at |

**Шаг 5: Подключите credentials**

- GOOGLE_SHEETS_CREDENTIALS_ID
- GOOGLE_DRIVE_CREDENTIALS_ID
- CREATOMATE_CREDENTIALS_ID
- TELEGRAM_CREDENTIALS_ID

### Результат

- Видео рендерится в Creatomate (~2-5 мин)
- Превью отправляется в Telegram
- Статус меняется на `ready_to_publish`
- URL видео сохраняется в `final_video_url`

6. Отправляет уведомление в Telegram
7. Ждёт и отправляет готовое видео
8. Обновляет статус в Sheets

## Workflow 6: Threads — Короткие посты

**Файл:** `workflow-6-threads-short-posts.json`

**Назначение:** Автоматическая публикация коротких постов в Threads на основе новостей

### Что делает

1. Каждые 2 часа читает `News Database` из Google Sheets
2. Фильтрует pending новости
3. Сортирует по score (приоритет)
4. Читает историю постов для контекста стиля
5. Читает аналитику для улучшения контента
6. **AI Agent (Claude):**
   - Генерирует короткий пост (150-280 символов)
   - Учитывает стиль предыдущих постов
   - Делает пост провокационным/вовлекающим
7. Публикует в Threads через API (2-шаговый процесс)
8. Обновляет статус новости в Sheets
9. Сохраняет пост в `Published Posts`
10. Отправляет уведомление в Telegram

### Структура workflow

```
Schedule Trigger (каждые 2ч)
├── Ветка 1: News Database → Filter → Sort by score → Published Posts → Code (подготовка)
└── Ветка 2: Get prompt file → Extract → Analytics Insights → Limit → Aggregate
                    ↓
              AI Agent (Claude 3.5 Sonnet)
                    ↓
         HTTP Request (create thread)
                    ↓
              Wait (30 сек)
                    ↓
         HTTP Request1 (publish)
                    ↓
         Google Sheets (update status)
                    ↓
         Append row (Published Posts)
                    ↓
              Telegram
```

### Настройка

**Шаг 1: Получите Threads API токен**

1. Зайдите на developers.facebook.com
2. Создайте приложение → тип "Business"
3. Добавьте продукт "Threads API"
4. Настройте разрешения:
   - `threads_basic`
   - `threads_content_publish`
5. Сгенерируйте Access Token (долгосрочный)

**Шаг 2: Замените ID аккаунта Threads**

В нодах `HTTP Request` и `HTTP Request1`:
```
24902864969393604 → ваш Threads User ID
```

Получить ID:
```
GET https://graph.threads.net/v1.0/me?access_token=YOUR_TOKEN
```

**Шаг 3: Создайте Google таблицу**

**Лист 1: News Database**
| id | title | summary | url | score | status | published_at |

**Лист 2: Published Posts**
| content | published_at | news_id |

**Лист 3: Analytics Insights**
| insight | value | date |

**Шаг 4: Замените ID таблицы**

Во всех нодах Google Sheets:
```
1ZHqRFOud0BhJB1BMpfBlpiUH2vae9go5CTohWGAWSqo → ваш ID таблицы
```

**Шаг 5: Настройте промпт-файл (опционально)**

В ноде `Get a file` можно указать URL к файлу с промптом:
```
https://raw.githubusercontent.com/YOUR_USERNAME/ai-content-prompts/main/short-post-prompt.txt
```

Или удалить эту ветку и использовать встроенный промпт в AI Agent.

**Шаг 6: Замените Telegram Chat ID**

В ноде `Telegram`:
```
YOUR_TELEGRAM_CHAT_ID → ваш ID чата
```

**Шаг 7: Подключите credentials**

- Google Sheets OAuth2
- Anthropic API (Claude)
- Threads API (Header Auth)
- Telegram Bot

### Как работает публикация в Threads

Threads API использует двухшаговую публикацию:

1. **Создание контейнера** (HTTP Request):
```
POST https://graph.threads.net/v1.0/{user_id}/threads
{
  "media_type": "TEXT",
  "text": "Текст поста"
}
```
Возвращает `creation_id`

2. **Публикация** (HTTP Request1):
```
POST https://graph.threads.net/v1.0/{user_id}/threads_publish
{
  "creation_id": "..."
}
```

Между шагами нужна пауза 30 сек (Wait node).

### Результат

- Каждые 2 часа публикуется новый пост в Threads
- Посты генерируются на основе свежих новостей
- AI учитывает аналитику и историю постов
- Статус новости меняется на `published`
- Уведомление приходит в Telegram

## Troubleshooting

**Apify не парсит:**
- Проверьте токен
- Возможно rate limit — подождите час

**OpenRouter ошибка:**
- Проверьте баланс
- Модель может быть недоступна — попробуйте другую

**HeyGen долго генерирует:**
- Нормально для видео 1-2 мин — до 10 минут
- Проверьте статус в личном кабинете

**Creatomate ошибка:**
- Проверьте формат URL видео/аудио
- Убедитесь что шаблон содержит все нужные элементы

**Google Drive не скачивает:**
- Проверьте права доступа к папкам
- Файлы должны быть доступны по ссылке
