# 🤖 AI Content Automation System - Автономная система контент-маркетинга

> **Самообучающаяся система для автоматизации контента в социальных сетях с интеграцией Claude Code + GitHub + n8n**

## 🎯 Результаты

- **1.000.000** просмотров в месяц
- **10-20** постов в день на полном автомате
- **0 минут** ежедневной работы (только 10 минут в неделю на мониторинг)
- **Автоматическая адаптация** на основе реальных данных

## 🔥 Что делает эту систему уникальной?

Это НЕ просто "ChatGPT пишет текст → копируешь в соцсеть". Это полноценная **автономная система**, которая:

✅ **Полный цикл** от мониторинга трендов до публикации
✅ **Самообучение** на основе реальной статистики
✅ **Автоматические обновления** логики через GitHub
✅ **Адаптация** под результаты БЕЗ вашего участия
✅ **Мультимодальность** (тексты, картинки, форматирование)

## 🏗️ Архитектура системы

Система состоит из 7 связанных workflow, которые работают автономно:

```
┌─────────────────────────────────────────────────────────────┐
│  1. Обновление токенов (OAuth)                              │
│     ↓ каждые 12 часов                                       │
│  2. Мониторинг новостей (RSS + LLM фильтрация)             │
│     ↓ каждые 6 часов                                        │
│  3. Генерация коротких постов (GitHub Prompts)              │
│     ↓ каждые 4 часа                                         │
│  4. Генерация каруселей (высокий engagement)                │
│     ↓ 1 раз в день                                          │
│  5. Публикация + сбор аналитики в реальном времени          │
│     ↓ каждые 2 часа                                         │
│  6. GitHub Sync (автообновление промптов)                   │
│     ↓ при пуше в репозиторий                                │
│  7. ML Анализ + Самообучение                                │
│     ↓ каждые 6 часов                                        │
└─────────────────────────────────────────────────────────────┘
```

## 📦 Workflow файлы

### [01-token-refresh.json](01-token-refresh.json)
**Обновление токенов OAuth**
- Автоматическое обновление access token каждые 12 часов
- Сохранение в Google Sheets для использования другими workflow
- Предотвращение разрыва интеграций

### [02-news-monitoring.json](02-news-monitoring.json)
**Мониторинг и фильтрация новостей**
- Парсинг RSS-фидов из заранее выбранных источников
- LLM фильтрация (Claude) - что важно, что нет
- Оценка потенциала вовлечения
- Сохранение в Google Sheets для дальнейшей обработки
- Уведомления в Telegram

### [03-short-posts.json](03-short-posts.json)
**Генерация коротких постов**
- Чтение промптов из GitHub (автосинхронизация)
- Выбор лучших паттернов на основе статистики
- Генерация постов через Claude
- AI-оптимизация текста
- Сохранение в очередь на публикацию

### [04-carousel-posts.json](04-carousel-posts.json)
**Генерация карусельных постов**
- Отбор новостей с высоким potential engagement
- Создание 10-слайдовых каруселей
- Генерация визуальных подсказок для дизайна
- Система одобрения через Telegram
- Автоматическая публикация после approve

### [05-full-posts-publishing.json](05-full-posts-publishing.json)
**Публикация постов + аналитика**
- Оптимальное время публикации (на основе ML)
- Публикация в Threads API
- Webhook для получения аналитики в реальном времени
- Анализ эффективности паттернов
- Автоматическая адаптация стратегии

### [06-github-sync.json](06-github-sync.json)
**Синхронизация с GitHub**
- Webhook на push в репозиторий
- Автоматическое обновление промптов в n8n
- Ежедневный бэкап статистики в GitHub
- Версионирование конфигураций

### [07-analytics-adaptation.json](07-analytics-adaptation.json)
**Самообучающийся алгоритм**
- Сбор статистики из Threads API
- ML-анализ всех постов
- Определение лучших паттернов
- Автоматическое обновление весов паттернов
- Оптимизация времени публикации
- Генерация новых конфигов и пуш в GitHub

## 🚀 Установка и настройка

### Требования

- **n8n** (self-hosted или cloud)
- **Google Sheets** API
- **Threads API** (или другая соцсеть)
- **GitHub** репозиторий
- **Telegram Bot** (для уведомлений)
- **Claude API** (Anthropic)

### Шаг 1: Клонирование

```bash
git clone <your-repo>
cd ai-content-automation
```

### Шаг 2: Импорт workflow в n8n

1. Откройте n8n
2. Для каждого файла `01-token-refresh.json` ... `07-analytics-adaptation.json`:
   - Нажмите "Import from File"
   - Выберите файл
   - Сохраните workflow

### Шаг 3: Настройка переменных окружения

В n8n создайте следующие environment variables:

```bash
# Google Sheets
GOOGLE_SHEET_ID=your_sheet_id

# Threads API
THREADS_API_URL=https://graph.threads.net/v1.0
THREADS_USER_ID=your_user_id

# GitHub
GITHUB_OWNER=your_username
GITHUB_REPO=ai-content-prompts
GITHUB_PROMPTS_URL=https://raw.githubusercontent.com/your_username/ai-content-prompts/main

# OAuth
OAUTH_TOKEN_URL=your_oauth_url
CLIENT_ID=your_client_id
CLIENT_SECRET=your_client_secret
REFRESH_TOKEN=your_refresh_token

# Telegram
TELEGRAM_MONITORING_CHAT_ID=your_chat_id
TELEGRAM_REVIEW_CHAT_ID=your_review_chat_id
```

### Шаг 4: Структура Google Sheets

Создайте Google Sheet с следующими листами:

1. **tokens** - для хранения OAuth токенов
2. **news_sources** - список RSS источников
3. **filtered_news** - отфильтрованные новости
4. **generated_posts** - сгенерированные посты
5. **carousel_posts** - карусельные посты
6. **post_statistics** - статистика постов
7. **ml_insights** - результаты ML анализа
8. **github_prompts** - промпты из GitHub

### Шаг 5: Создание GitHub репозитория для промптов

```bash
mkdir ai-content-prompts
cd ai-content-prompts
mkdir prompts config backups

# Создайте файлы промптов
touch prompts/short-posts.json
touch prompts/carousel-posts.json
touch config/active-config.json

git init
git add .
git commit -m "Initial commit"
git push
```

Пример `prompts/short-posts.json`:

```json
{
  "provocative": {
    "template": "Провокационный вопрос + неожиданный факт + призыв к действию",
    "style": "Прямолинейный, вызывающий дискуссию",
    "example": "Все говорят про AI агентов, но 90% даже не понимают, что это такое. А вы?"
  },
  "educational": {
    "template": "Проблема + решение + call to action",
    "style": "Образовательный, практичный",
    "example": "Хотите автоматизировать контент? Вот 3 инструмента, которые работают."
  },
  "storytelling": {
    "template": "История + инсайт + вывод",
    "style": "Нарративный, эмоциональный",
    "example": "Потратил 3 месяца на создание AI системы. Вот что получилось."
  }
}
```

### Шаг 6: Настройка webhook в GitHub

1. Откройте ваш репозиторий на GitHub
2. Settings → Webhooks → Add webhook
3. Payload URL: `https://your-n8n-instance.com/webhook/github-prompts-update`
4. Content type: `application/json`
5. Events: `Push`

### Шаг 7: Активация workflow

1. Откройте каждый workflow в n8n
2. Нажмите "Activate" в правом верхнем углу
3. Проверьте, что все credential'ы настроены

## 🎯 Как это работает?

### 1️⃣ Сбор новостей

Система автоматически мониторит заранее выбранные источники (RSS-фиды) каждые 6 часов:

- Парсит статьи
- Извлекает контент (HTML → текст)
- Фильтрует через Claude LLM
- Оценивает потенциал engagement
- Сохраняет в Google Sheets

### 2️⃣ Генерация постов

Каждые 4 часа система:

1. Читает промпты из GitHub
2. Анализирует статистику прошлых постов
3. **Выбирает лучшие паттерны** (те, что показали высокую эффективность)
4. Генерирует новые посты через Claude
5. Сохраняет в очередь

**Магия:** Система сама видит "ага, провокационные посты дают в 3 раза больше охвата" и начинает их генерить больше.

### 3️⃣ Публикация

Каждые 2 часа система:

1. Анализирует статистику и определяет **оптимальное время**
2. Публикует посты в Threads
3. Сохраняет post_id для отслеживания

### 4️⃣ Аналитика в реальном времени

Через webhook от Threads система получает:

- Просмотры
- Лайки, комментарии, шеры
- Engagement rate
- Virality score

### 5️⃣ Самообучение (ключевая фича!)

Каждые 6 часов запускается ML-алгоритм:

```javascript
// Анализирует ВСЮ статистику
// Группирует по паттернам
// Вычисляет success_score для каждого паттерна
// Определяет:
//   - Какие паттерны усилить (frequency_multiplier > 1)
//   - Какие ослабить (frequency_multiplier < 1)
//   - Лучшее время для постинга
//   - Оптимальную длину постов

// Генерирует новую конфигурацию
// Пушит в GitHub
// GitHub webhook → n8n автоматически обновляет промпты
```

### 6️⃣ Автоматическое обновление через GitHub

Когда вы (или система) вносите изменения в промпты:

1. `git push` в репозиторий
2. GitHub webhook триггерит n8n
3. n8n скачивает новые промпты
4. Обновляет Google Sheets
5. Система начинает использовать новые промпты **без перезапуска**

## 📊 Результаты и метрики

### Ключевые метрики

- **Engagement Rate**: (likes + comments×2 + shares×3 + saves×2) / views × 100
- **Virality Score**: shares / views × 1000
- **Retention Score**: saves / views × 100
- **Success Score**: взвешенная формула из всех метрик

## 🔧 Кастомизация

### Изменение промптов

Просто отредактируйте файлы в `prompts/` и сделайте push:

```bash
cd ai-content-prompts
nano prompts/short-posts.json
git add .
git commit -m "Update prompts"
git push
```

Система автоматически подхватит изменения.

### Добавление новых паттернов

1. Добавьте новый паттерн в `prompts/short-posts.json`
2. Пуш в GitHub
3. Система начнет тестировать новый паттерн
4. Через несколько дней ML-алгоритм оценит его эффективность
5. Если паттерн работает - система начнет использовать его чаще

### Интеграция с другими соцсетями

Измените workflow 05:
- Замените Threads API на нужный (Twitter, LinkedIn, Instagram)
- Обновите поля в Google Sheets
- Настройте webhook для аналитики

## 🛠️ Troubleshooting

### Проблема: Промпты не обновляются из GitHub

**Решение:**
1. Проверьте webhook в GitHub (Settings → Webhooks)
2. Убедитесь, что URL webhook правильный
3. Проверьте логи в n8n (Workflow → Executions)

### Проблема: Посты не публикуются

**Решение:**
1. Проверьте OAuth токены в Google Sheets
2. Убедитесь, что workflow 01 (token refresh) активен
3. Проверьте Threads API credentials

### Проблема: Аналитика не собирается

**Решение:**
1. Убедитесь, что webhook от Threads настроен
2. Проверьте URL webhook в workflow 05
3. Проверьте формат данных от Threads

## 📈 Масштабирование

### Для увеличения количества постов:

1. Уменьшите интервалы в Schedule Trigger
2. Добавьте больше источников новостей в Google Sheets
3. Увеличьте `maxItems` в workflow 05 (Limit node)

### Для множественных аккаунтов:

1. Дублируйте workflow 03-05
2. Создайте отдельные листы в Google Sheets для каждого аккаунта
3. Настройте разные credentials для каждого аккаунта

## 🎓 Продвинутые возможности

### A/B тестирование

Добавьте в промпты два варианта одного паттерна:

```json
{
  "provocative_v1": { ... },
  "provocative_v2": { ... }
}
```

Система автоматически протестирует оба и выберет лучший.

### Сезонность

Создайте отдельные файлы промптов для разных периодов:

```
prompts/
  short-posts-winter.json
  short-posts-summer.json
  short-posts-holidays.json
```

Меняйте их вручную или настройте автоматическое переключение.

### Мультиязычность

Для каждого языка создайте свой workflow и свой набор промптов:

```
prompts/
  short-posts-en.json
  short-posts-ru.json
  short-posts-es.json
```
